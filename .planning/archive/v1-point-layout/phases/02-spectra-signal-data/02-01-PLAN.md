<objective>
Replace Phase 1's single-spectrum placeholder in `MzPeakSpectrumWriter` with the full
point-layout signal for ALL in-range spectra under DUAL representation: `spectra_data` (as-acquired
arrays for every in-range scan) plus a conditional `spectra_peaks` facet (Thermo CentroidStream label
peaks for profile scans that carry them). Extend the single-row `spectra_metadata` write into the
per-scan loop so it emits one `spectrum` row per output ordinal, keeping the data and metadata spectrum
sets identical. Canonical widths (mz f64, intensity f32), per-spectrum non-decreasing m/z with the
multiset preserved, honored filters, and `sorting_rank: 0` declared for `point.mz`.

Purpose: Make the spectral signal facets complete and correct (DATA-01..04) so Phase 3 can layer rich
metadata + the scan/precursor/selected_ion facets on top. Rich metadata fidelity stays out of scope
here; only the minimal per-ordinal spectrum row (index, id, time) is written so counts and the index
are internally consistent.
Output: An updated `MzPeakSpectrumWriter.cs` and a new `MzPeakWriterTests.cs`; converting `small.RAW`
yields a `spectra_data.parquet` covering all in-range spectra, a `spectra_metadata.parquet` with one row
per spectrum, and (since small.RAW carries profile scans with CentroidStream label peaks) a
`spectra_peaks.parquet`.
</objective>
