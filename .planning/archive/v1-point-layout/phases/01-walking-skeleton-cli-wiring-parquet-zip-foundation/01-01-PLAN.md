Wire a new `mzpeak` output format end-to-end through the TRFP CLI and emit the thinnest honest
archive that the reference Python reader can OPEN: a STORED ZIP containing a ZSTD-internal Parquet
facet pair (`spectra_data.parquet` + `spectra_metadata.parquet`) plus `mzpeak_index.json`, built

`MzPeakParquet` helper, and a reader-openable `<raw>.mzpeak`.

Parquet.Net path semantics (DO NOT string-split with '/'):
- A DataField's identity is the leaf object returned by schema.GetDataFields(). Pair each returned
  DataField with the caller-supplied array directly; never parse Path.ToString() with '/'.
- DataField.Path is a FieldPath whose ToString() is DOT-separated (e.g. "spectrum.mz_range.lo"),
  NOT '/'-separated. If a string key is ever needed, use the dot-separated FieldPath, but the write
  helper keys on the DataField objects themselves.
- DataColumn default codec is Snappy; ZSTD MUST be set explicitly on the writer and proven on read.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RAW file -> writer | Thermo RAW is parsed by trusted, already-shipping TRFP reader code (reused, not new) |
| writer -> .mzpeak archive | New code constructs Parquet + zip bytes consumed by external readers |
| .mzpeak -> reference reader | OPEN gate runs an external Python process over the produced file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | mzpeak_index.json lists a facet not actually written | mitigate | Emit JSON listing EXACTLY the two facets written; reader fails OPEN otherwise |
| T-01-02 | Denial of Service | Zero-point spectrum -> empty/invalid facet | mitigate | Select the first in-range spectrum with >= 1 real point; if none, throw RawFileParserException and emit no archive (never fabricate a point) |
| T-01-03 | Information Disclosure | Accidental deflate (CompressionLevel default) breaks STORED contract | mitigate | Force `CompressionLevel.NoCompression`; verify `compress_type==0` in the gate |
| T-01-04 | Tampering | Parquet codec silently left at Snappy default, violating PQ-03 | mitigate | Set `CompressionMethod.Zstd`; gate reads both parquet entries' column-chunk metadata and asserts ZSTD |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No new C# NuGet packages (Parquet.Net/Newtonsoft already deps). Only pip installs are `pynumpress`/`psims` into the local test env, verified on pypi.org during research; not shipped in output |
