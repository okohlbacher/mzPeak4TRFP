<row_model_note>
ROW MODEL (authoritative — RESEARCH.md Decision 1): `spectra_metadata.parquet` holds FOUR
co-resident top-level struct columns (`spectrum`, `scan`, `precursor`, `selected_ion`), each an
independent table laid out at row position = that table's own ordinal and right-padded with
all-null rows up to the shared file row count. ALL FOUR struct columns have N rows, where
N = spectrum count (= 48 for small.RAW). `spectrum`/`scan` are present (def-level 1) on all N
rows; `precursor`/`selected_ion` are present on rows 0..M-1 (M = count of emitted MSn ordinals)
and NULL (def-level 0) on rows M..N-1. Linkage is by the `source_index` field, NEVER by row
position. CONTEXT.md's earlier "each row populates exactly one struct, the other three are null"
(row-disjoint) wording is SUPERSEDED by this note.
</row_model_note>
