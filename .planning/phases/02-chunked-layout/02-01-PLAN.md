<objective>
Emit `spectra_data` in the reference chunk struct (lossless delta-encoded m/z) as the new default, with
`--point` restoring the v1 point layout. `spectra_peaks` and `chromatograms_data` stay point layout. Chunking
is a fixed m/z window over the sorted m/z axis (default 50.0, `--chunk-size` configurable), one chunk row per
non-empty window per spectrum, emitted through the Phase-1 streaming Handle as list columns. Phase 2 applies
NO zero-run nulling — chunked output decodes to the EXACT v1 point (m/z,intensity) multiset per spectrum.

Purpose: smaller, reference-structured spectra_data without regressing v1 conformance or losslessness; lays
the structure that Phase 3 (Numpress) and Phase 4 (ZRS null-marking) plug into.
Output: a chunk codec + window chunker, a `ChunkFacetStream`, a chunk `spectrum_array_index` footer, the
`--point`/`--chunk-size` CLI flags, and NUnit locks covering codec round-trip, multiset equality, schema diff,
validator 0/0 in both modes, and size shrink.
</objective>

  <name>Task 1: Chunk codec — null-aware delta encode/decode + fixed m/z-window chunker (no nulling in Phase 2)</name>
  <files>ThermoRawFileParser/Writer/MzPeak/MzPeakChunkCodec.cs, ThermoRawFileParser/ThermoRawFileParserTest/MzPeakChunkTests.cs</files>
  <behavior>
    Codec is null-aware in STRUCTURE (so Phase 4 ZRS can pass nulls later) but Phase 2 NEVER produces nulls.
    - DeltaEncode: given a window's m/z slice (no nulls in Phase 2), returns mz_chunk_start = first m/z,
      mz_chunk_end = last m/z, mz_chunk_values = consecutive deltas of length (k-1) — the start value is NOT
      in mz_chunk_values (it lives in mz_chunk_start). Mirror null_delta_encode: a value after a null is stored
      ABSOLUTE, a leading null is kept as a marker. With no nulls this reduces to [mz[1]-mz[0], mz[2]-mz[1],...].
    - DeltaDecode: given mz_chunk_start + mz_chunk_values, reconstructs the m/z array. Mirror null_delta_decode:
      start is the first reconstructed m/z; cumulative add for present deltas; absolute restart after a null;
      nulls preserved aligned to intensity. Reconstructed (mz,intensity) drops pairs where either is null.
    - Chunk(mz[], width): partition the sorted m/z axis into intervals each spanning <= width from the
      interval's own first m/z (threshold = first_mz + width, advance by width; mirror null_chunk_every_k,
      including: never emit a length-1 chunk — roll a singleton into the adjacent chunk; always emit the final
      residual interval). Returns interval [start,end) index boundaries; one interval per non-empty window.
    - Test 1 (round-trip, gapless): random sorted m/z + f32 intensity, chunk at 50.0, encode each window,
      decode, concatenate -> decoded (round(mz,6),intensity) multiset EQUALS input multiset, 0 mismatches.
    - Test 2 (duplicate m/z): input with repeated equal m/z survives encode->decode as the same multiset
      (deltas of 0.0 reconstruct exactly).
    - Test 3 (single-point spectrum): a 1-point m/z array yields one chunk, mz_chunk_values empty,
      mz_chunk_start==mz_chunk_end==the point; decode returns exactly that point.
    - Test 4 (null structural support): a hand-built window with an interior null in mz+intensity decodes with
      the null pair dropped and the post-null value treated as an absolute restart (proves Phase-4 readiness).
      This test constructs nulls directly; the writer path produces none in Phase 2.
  </behavior>
  <action>
