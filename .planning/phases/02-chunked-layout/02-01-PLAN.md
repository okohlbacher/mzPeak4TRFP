<objective>
Emit `spectra_data` in the reference chunk struct (delta-encoded m/z) as the new default, with `--point`
restoring the v1 point layout. Phase 2 chunks ONLY `spectra_data`: `spectra_peaks` stays point layout, and
`chromatograms_data` stays point layout as an EXPLICIT, deliberate deviation from the reference (see below).
Chunking is a fixed m/z window over the sorted m/z axis (default 50.0, `--chunk-size` configurable), one chunk
row per non-empty window per spectrum, emitted through the Phase-1 streaming Handle as list columns. Phase 2
applies NO zero-run nulling — chunked output decodes to the v1 point (m/z,intensity) multiset per spectrum,
verified BITWISE (CHUNK-06; see verification for the exact/bounded decision procedure).

Scope deviation (P1, documented): the authoritative reference `refs/mzPeak/small.chunked.mzpeak` emits
`chromatograms_data` in CHUNK layout (time_chunk_start/end/values, chunk_encoding=MS:1003089, intensity,
ms_level) — NOT point. Earlier CONTEXT/RESEARCH wrongly claimed the reference keeps chromatograms point; that
claim is FALSE and is corrected here. Phase 2 deliberately keeps `chromatograms_data` POINT because the
validator has no chromatogram structural rule and the TIC is tiny (chunking yields negligible size benefit).
Full chromatogram chunking for complete structural parity is DEFERRED to Phase 5 (or a tracked follow-up).

Purpose: smaller, reference-structured spectra_data without regressing v1 conformance or losslessness; lays
the structure that Phase 3 (Numpress) and Phase 4 (ZRS null-marking) plug into.
Output: a chunk codec + window chunker (with empty/all-equal/non-monotonic/singleton guards and a reference
null-aware decoder), a `ChunkFacetStream`, a chunk `spectrum_array_index` footer, the `--point`/`--chunk-size`
CLI flags, and NUnit locks covering codec round-trip, edge cases, reference-null decode, bitwise multiset
equality, footer-parse lock, schema diff, validator 0/0 in both modes, and size shrink.
</objective>

    - DeltaEncode: given a window's m/z slice (no nulls in Phase 2), returns mz_chunk_start = first m/z,
      mz_chunk_end = last m/z, mz_chunk_values = consecutive deltas of length (k-1) — the start value is NOT
      in mz_chunk_values (it lives in mz_chunk_start). Mirror null_delta_encode: a value after a null is stored
      ABSOLUTE, a leading null is kept as a marker. With no nulls this reduces to [mz[1]-mz[0], mz[2]-mz[1],...].
    - DeltaDecode: given mz_chunk_start + mz_chunk_values, reconstructs the m/z array. Mirror null_delta_decode
      byte-for-byte: start is the first reconstructed m/z; cumulative add for present deltas; absolute restart

    - Chunk(mz[], width): partition the sorted m/z axis into intervals each spanning <= width from the

      residual interval). Returns interval [start,end) index boundaries; one interval per non-empty window.
    - Edge-case contract (P4), defined exactly and tested:
      * EMPTY spectrum (k==0): Chunk returns zero intervals; the writer emits NO chunk row (mirror reference —
        no empty/all-null chunk rows).
      * ALL-EQUAL m/z chunk: every delta is 0.0; encode->decode reproduces the identical repeated value exactly.
      * DESCENDING / NON-MONOTONIC m/z: the codec ASSUMES ascending sorted input (the per-scan path sorts).
        Guard: if input is not non-decreasing, the chunker/encoder throws an explicit ArgumentException
        (it does NOT silently produce a wrong layout). Document this contract; do not attempt to re-sort here.
      * SPARSE BOUNDARY SINGLETON: a window that would close with a single point is rolled into the adjacent
        chunk (length-1 chunks never emitted); the final residual singleton is folded into the previous chunk.
    - Test 1 (round-trip, gapless): random sorted m/z + f32 intensity, chunk at 50.0, encode each window,

      bitwise multiset (deltas of 0.0 reconstruct exactly).

      mz_chunk_start==mz_chunk_end==the point; decode returns exactly that point.
    - Test 4 (empty + non-monotonic guards): empty input yields zero intervals and no chunk row; a
      descending/non-monotonic input throws ArgumentException.
    - Test 5 (reference null decode [None, absolute, delta]): hand-built mz_chunk_values starting
