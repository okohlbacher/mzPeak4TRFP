<objective>
Add Numpress-linear m/z compression to the chunked spectra_data layout and make it the new default.
A vendored, pure-managed C# MSNumpress port encodes each chunk's sorted m/z into a `mz_numpress_linear_bytes`
list<uint8> column (7th chunk field), `mz_chunk_values` becomes null, `chunk_encoding = MS:1002312`, and the
intensity list stays the plain lossless `large_list<float>`. `--no-numpress` / `--lossless` falls back to the
Phase-2 delta chunks (bitwise-L1); `--point` falls back to the v1 point layout (bitwise-L1).

LOCKED scope: intensity stays lossless f32. We do NOT implement intensity Numpress-SLOF (the reference does;
that drives its ~63% total reduction). Our numpress output is therefore a VALID m/z-only-numpress variant of
the chunk schema, intentionally NOT byte/schema-identical to small.numpress.mzpeak. Intensity-SLOF is backlog.

Purpose: smallest m/z encoding (the size win), self-describing transform, bounded L2 m/z fidelity while
intensity remains exact. Output: the MSNumpress codec + numpress chunk emission + CLI default + L2/3-mode
verification.
</objective>

ChunkFacetStream.Append (MzPeakSpectrumWriter.cs ~963) builds one chunk row per window: it computes the
window slice, calls MzPeakChunkCodec.DeltaEncode -> start/end/values, sets _bEnc.Add(ChunkEncodingCurie),
emits mz_chunk_values (delta) + intensity. The numpress branch keeps start/end (true first/last m/z anchors),
sets encoding = NumpressLinearCurie, emits mz_chunk_values as an EmptyList/null row, and emits the numpress
bytes row from MSNumpress.EncodeLinear(slice-as-double[]).

ChunkFacetStream.Flush (MzPeakSpectrumWriter.cs ~1007) writes columns via NestedLevels + WriteRowGroupAsync.
A numpress byte-list leaf is added the same way mz/intensity item leaves are (Leaf + ListOf + NestedLevels).

<risks>
- **Numpress port correctness / L2 bound (chief):** the residual 4-bit half-byte varint and 5-byte LE fixed-point packing are the easiest places to mis-port. Mitigated by the ms-numpress known vector (byte-identity), pynumpress byte cross-check at a shared fp, and the per-chunk 0.5/fp decode assertion. The phantom-leading-value decode artifact is handled in the codec's anchor-aligned decode helper, not the raw codec.
- **Reintroducing an x64 dependency:** port uses System only; AnyCPU/MSIL assembly assertion guards against an accidental native pull-in.
- **Keeping --lossless/--point bit-exact:** numpress is gated behind a flag and a separate struct/footer; the delta path and point path are untouched, and bitwise-L1 locks fail loudly if either regresses.
- **CURIE discrepancy:** ROADMAP says MS:1003089 for numpress; verified dump says MS:1002312. We follow the verified artifact and document the override.
- **Size win is m/z-only:** without intensity SLOF the reduction is the m/z saving (not the reference's ~63%). The size lock asserts numpress < delta-chunked (the achievable target), not the reference's headline.
</risks>
