# Phase 4 (rescoped): Ion Mobility (FAIMS)

Profile compaction (zero-run stripping + null-marking + δmz model, ZRS-01..04) is DROPPED to backlog
BL-02 by decision (high complexity/fidelity-risk for the size benefit). This plan covers only the
ion-mobility gap: populate the FAIMS columns (which already exist, currently null).

## Background (from RESEARCH.md item 5, verified)
- `ScanStager` already builds the `ScanTrailer` per scan. FAIMS is read via `trailer.AsBool("FAIMS Voltage On:")`
  (gate) + `trailer.AsDouble("FAIMS CV:")` (the compensation voltage). The `scan.ion_mobility_value` (double)
  and `scan.ion_mobility_type` (string) columns already exist (emitted null since the review's Wave 1).
- CURIE for the type = **MS:1001581** "FAIMS compensation voltage".
- `small.RAW` has NO FAIMS → columns stay null (correct); validate against a SYNTHETIC ScanTrailer in tests.

## Tasks

### Task 1 — populate FAIMS ion-mobility in ScanStager
- In `ScanStager`, when staging a scan: if `trailer.AsBool("FAIMS Voltage On:")` is true AND `AsDouble("FAIMS CV:")`
  yields a value, set the staged record's `IonMobilityValue` (double?) = that CV and `IonMobilityType` =
  the CURIE `MS:1001581` (add to `MzPeakCv`). Otherwise leave both null (current behavior).
- The metadata builder already emits `scan.ion_mobility_value`/`ion_mobility_type` leaves — wire the record
  values through (currently hard-null). Add `MS:1001581` to the generated `cv_list` ONLY when emitted
  (reuse the CvCollector so cv_list stays exhaustive and doesn't drift).
- IM-02: `selected_ion` ion-mobility — the reference leaves it null unless applicable; keep null (no Thermo
  source for selected-ion mobility here). Document.
- **Acceptance:** builds; converting small.RAW (no FAIMS) keeps ion_mobility_* null and cv_list unchanged;
  mzpeak-validate 0/0.

### Task 2 — tests (synthetic FAIMS, since small.RAW has none)
- Unit test driving the FAIMS extraction with a SYNTHETIC `ScanTrailer` (or a small seam): "FAIMS Voltage On:"=true
  + "FAIMS CV:"=-45.0 → record.IonMobilityValue == -45.0, IonMobilityType == "MS:1001581", and (if emitted into
  a facet) cv_list contains MS:1001581. Negative case: FAIMS off → both null, MS:1001581 NOT in cv_list.
- Confirm small.RAW conversion: scan.ion_mobility_value/type all null (pyarrow), and the 3-mode mzpeak-validate
  stays 0/0 (numpress/--lossless/--point).
